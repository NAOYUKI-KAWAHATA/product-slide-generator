# 商品スライド制作スキル v2

## 概要

講座型商品の販売用スライド資料を、**Manusスライド機能（画像生成モード）** で52〜66ページ生成する。全ページを1枚ずつAI画像として生成し、Canvaテンプレートをリファレンスとした洗練された高級感を実現する。

**デザインの2大ルール（必ず守ること）:**
1. **見出しバナーは全コンテンツページで完全統一** — 同じ形・色・フォントのバナーを使う
2. **ロゴ・ブランドマークは一切表示しない** — 「M」ロゴ含め全て非表示

---

## Phase 1: 商品概要の受け取りとヒアリング

### Step 1: 初回案内メッセージ

スキル起動時に以下を送信する。

> 講座スライド資料を作成します！
> まずは**商品・サービスの概要**を教えてください。
>
> 「商品設計プランナースキル」で作成した商品設計書やスライド下書きがあれば、そのまま貼り付けていただくのが一番スムーズです。
>
> もしくは、以下の情報を教えてください：
>
> 1. **講座名**（キャッチコピー）
> 2. **コンセプト**（誰を、どんな未来に導くか）
> 3. **ターゲット像**（年齢・職業・悩み）
> 4. **Before / After**（受講前の悩み → 受講後の理想）
> 5. **カリキュラム**（章立て・各章の内容）
> 6. **提供コンテンツ**（動画教材・テンプレート・グループ講義など）
> 7. **料金プラン**（スタンダード / VIP など）
> 8. **特典**（参加特典・VIP限定特典）
> 9. **講師プロフィール**（肩書き・実績）
> 10. **保証・支払い方法**

### Step 2: 受け取った情報の確認

ユーザーから情報を受け取ったら、以下を確認する。

| 確認項目 | 必須/任意 | 不足時の対応 |
|---|---|---|
| 講座名 | 必須 | ヒアリングする |
| コンセプト | 必須 | ヒアリングする |
| ターゲット像 | 必須 | ヒアリングする |
| Before / After | 必須 | ヒアリングする |
| カリキュラム（6章） | 必須 | ヒアリングする |
| 提供コンテンツ | 必須 | カリキュラムから推測して提案 |
| 料金プラン | 必須 | ヒアリングする |
| 特典 | 任意 | なければスキップ（該当ページ削除） |
| 講師プロフィール | 必須 | ヒアリングする |
| 保証・支払い方法 | 任意 | なければデフォルト提案 |

不足情報がある場合は**まとめて1回で**ヒアリングする。2回以上に分けない。

### Step 3: スライド構成の生成

全情報が揃ったら、`slide_structure_template.md` を参照してスライド構成（全ページのP#・レイアウト種別・記載内容）を内部的に生成する。

```
READ: /home/ubuntu/skills/product-design-planner/references/slide_structure_template.md
```

提供コンテンツの枚数（5〜10枚）と特典の枚数はユーザーの商品内容に応じて増減する。総ページ数は52〜66ページの範囲に収める。

## Phase 6: スライド生成

### Step 1: プロジェクト初期化

`slide_initialize` で画像生成モードのプロジェクトを作成する。

- `generate_mode`: `image`
- `height_constraint`: `true`（16:9）
- `project_dir`: `/home/ubuntu/product_slides`

**アウトラインの作成ルール**: Phase 1で生成したスライド構成の全ページを `outline` に変換する。各ページの `id` はレイアウト種別＋連番（例: `cover_01`, `statement_01`, `bullet_list_01`）とする。

### Step 2: ページごとの画像生成

全ページを順番に `image_slide_generate` で生成する。

**生成ルール（全ページ共通）:**

1. `design_system.md` の共通デザイン指示を**毎回プロンプトの末尾に付加する**
2. 各ページのレイアウトに対応するCanvaリファレンス画像を `references` の**最初の要素**として含める
3. 2ページ目以降は、直前に生成した画像も `references` に含める（最大2枚）
4. レイアウト種別ごとのプロンプトテンプレートは `layout_prompts.md` を参照する

**referencesの指定例（P3以降）:**
```
references: [
  "/home/ubuntu/skills/product-slide-generator/templates/canva_reference/B_bullet_list.jpg",  // Canvaテンプレート（最優先）
  "/home/ubuntu/product_slides/p02_generated.webp"  // 直前ページ（デザイン継続性）
]
```

**見出しバナーの統一確認**: 生成前に必ず確認する。
- コンテンツページ: 左上に「ダークチャコール(#333333)バナー＋白文字の見出し」を表示
- 表紙・セクション扉・インパクト文ページ: バナーなし
- ロゴ・ブランドマーク: 一切表示しない

### Step 3: 進捗報告

10ページごとにユーザーに進捗を報告する。

> 現在 {{完了数}} / {{総数}} ページ完了しました。

### Step 1: 全ページの確認

生成完了後、`slide_present` でスライドを表示する。

### Step 2: ユーザーへの納品メッセージ

> スライド資料が完成しました！（全{{総数}}ページ）
>
> **デザイン**: モノトーン＋シャンパンゴールド
>
> 修正したいページがあれば、ページ番号と修正内容を教えてください。
> 例: 「P4のプロフィール写真を差し替えたい」「P15の文字を修正したい」

### Step 3: 修正対応

ユーザーから修正依頼があった場合、該当ページのみ `image_slide_generate` で再生成する。そのページのCanvaリファレンス画像と前後のページ画像を `references` に含め、デザインの一貫性を保つ。

## 注意事項

### 見出しバナーの一貫性（最重要）

全コンテンツページで見出しバナーのデザインが統一されていることを確認する。バナーの形・色（#333333）・フォントが1ページでも異なると品質NG。

### 日本語テキストの精度

画像生成モードでは日本語テキストが崩れやすい。以下の対策をプロンプトに必ず含める。

- 「Render all Japanese text EXACTLY as written, no missing or garbled characters」
- 長い文章は短いフレーズに分割して指示する
- 漢字・ひらがな・カタカナの混在を明示する

### ページ数の増減

スライドのページ数は52〜66ページの範囲で変動する。提供コンテンツや特典の数に応じて自動的に調整される。

### 生成の中断と再開

生成途中でエラーが発生した場合、最後に成功したページから再開する。`slide_state.json` の `state` フィールドで各ページの生成状況を確認できる。
